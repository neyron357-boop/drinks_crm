# Учет продаж напитков

MVP веб/PWA-приложения для ежедневного учета продаж напитков по точкам доставки.

## Что реализовано

- ежедневный отчет по дате, точке и водителю;
- автоматический расчет продажи, суммы и заявки;
- приход по позициям без влияния на продажи сверх формулы;
- перемещения между точками со списанием у отправителя и приходом у получателя;
- касса водителя с формулой "Должен сдать" и "Недостача / Плюс";
- закрытие дня с блокировкой отчета;
- роли: админ, менеджер, водитель, бухгалтер;
- dashboard по продажам, точкам, водителям, товарам, расходам, долгам и недостачам;
- Excel-экспорт: листы по точкам, "ЗАЯВКА", "ДОЛГИ", день/период, одна/все точки;
- PWA manifest и service worker;
- локальное автосохранение в браузере.

## Запуск

```bash
npm install
npm run dev
```

Локально: [http://127.0.0.1:3000](http://127.0.0.1:3000)

Проверки:

```bash
npm run typecheck
npm run build
```

## Backend

Текущая версия работает локально через `localStorage`, чтобы можно было быстро проверить рабочий процесс без ключей Supabase.

Supabase/PostgreSQL миграции лежат в [supabase/migrations](./supabase/migrations). Инструкция по привязке проекта, ролям, RLS и переносу остатков: [SUPABASE_SETUP.md](./SUPABASE_SETUP.md).

## Примечание по аудиту зависимостей

`npm audit` сейчас показывает moderate-предупреждения в транзитивных зависимостях `next -> postcss` и `exceljs -> uuid`. Автоматический `npm audit fix --force` предлагает breaking downgrade, поэтому я не применял его.
